# First Steps: Docker VM with Basic Profile

This guide takes you from a fresh clone to a running Ubuntu VM with the basic
profile applied and Claude Code ready to use. It uses the Docker provider,
which works on any Linux host with a Docker daemon — no cloud account needed.

**Estimated time**: 20–30 minutes.

## Prerequisites

| Requirement | How to install |
|---|---|
| Docker daemon (running) | [docs.docker.com/get-docker](https://docs.docker.com/get-docker/) |
| Current user in `docker` group | `sudo usermod -aG docker $USER`, then re-login |
| Python 3, pip, and `python3-venv` | `sudo apt install python3-pip python3-venv` (Debian/Ubuntu) |
| `sshpass` | `sudo apt install sshpass` (Debian/Ubuntu) |

## 1. Install Python Dependencies and Ansible Collections

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
ansible-galaxy collection install -r requirements.yml
```

> [!NOTE]
> Activate the virtualenv (`. .venv/bin/activate`) at the start of every new
> shell session before running Ansible commands.

## 2. Generate an SSH Key

Skip this step if you already have `~/.ssh/id_ansible_ed25519`.

```bash
ssh-keygen -t ed25519 -f ~/.ssh/id_ansible_ed25519 -C "$(whoami)@$(hostname)"
```

Load it into your SSH agent:

```bash
ssh-add ~/.ssh/id_ansible_ed25519
```

If you see "Could not open a connection to your authentication agent", start
one first:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ansible_ed25519
```

> [!NOTE]
> The playbooks verify SSH connectivity using this key. Without it in the
> agent, step 4 fails.

## 3. Configure Vault Secrets

Playbooks read secrets from an Ansible Vault-encrypted file at
`inventories/group_vars/all/vault.yml`. Set it up once:

```bash
# Choose a vault password and persist it
echo -n "Vault password: " && read -rs ANSIBLE_VAULT_PASSWORD && echo
printf 'export ANSIBLE_VAULT_PASSWORD=%q\n' "$ANSIBLE_VAULT_PASSWORD" >> .envrc
. .envrc

# Create and encrypt the secrets file from the provided template
cp inventories/group_vars/all/vault-template.yml \
   inventories/group_vars/all/vault.yml
ansible-vault encrypt inventories/group_vars/all/vault.yml

# Open the encrypted file in your editor
ansible-vault edit inventories/group_vars/all/vault.yml
```

Fill in these fields:

| Field | Value |
|---|---|
| `vault_my_ansible_user_password` | Password for the `gandalf` admin account |
| `vault_my_ssh_public_key` | Output of `cat ~/.ssh/id_ansible_ed25519.pub` |
| `vault_galadriel.password` | Login password for the `galadriel` desktop account |
| `vault_galadriel.exa_api_key` | [Exa](https://exa.ai) API key (any placeholder string works for local use) |

> [!NOTE]
> Cloud-specific fields (`vault_my_ssh_key_name`, `vault_windows_admin_password`,
> etc.) can stay as placeholder strings for a Docker-only setup.
>
> `.envrc` is gitignored. Future shell sessions need `ANSIBLE_VAULT_PASSWORD`
> re-exported. Consider [direnv](https://direnv.net) to automate this.

See [Important Concepts](./important-concepts.md) for background on the user
model and vault setup.

## 4. Create the VM

```bash
ansible-playbook playbooks/create-vm.yml --extra-vars provider=docker
```

On first run this builds the `ansible-vm-docker:latest` container image (a few
minutes); subsequent runs reuse it. The playbook prints the assigned hostname
on completion — for example, `tatooine`.

> [!NOTE]
> If this step fails, the playbook cleans up the container automatically.
> Fix the problem and re-run the same command.

## 5. Apply the Basic Profile

```bash
ansible-playbook playbooks/configure-profile.yml
```

No `--extra-vars provider=docker` flag is needed here: `ansible.cfg`
auto-loads all files under `./inventories/`, so `configure-profile.yml` picks
up the Docker host that step 4 registered.

This takes around 10 minutes. On completion:

- The `gandalf` admin account and the `galadriel` desktop account exist with
  sudo privileges and SSH key access; password SSH login is disabled.
- OS packages are up to date and the timezone is set to `Europe/Berlin`.
- Installed tools include Podman, tmux, Ruby, Python, Dolt SQL Server,
  Claude Code, and OpenCode.

## 6. Log In as the Desktop User

Find the dynamic SSH port assigned to the container:

```bash
grep ansible_port inventories/docker_autogenerated.yml
```

Connect with your SSH key:

```bash
ssh -p <ansible_port> -i ~/.ssh/id_ansible_ed25519 galadriel@127.0.0.1
```

## 7. Start Claude Code

```bash
claude
```

Claude Code is installed but not yet authenticated. On first launch it prints
a URL and a one-time code. Open the URL in a browser on your **host** machine
(the container has no display), complete the login, then paste the code back
into the terminal prompt.

## Tear Down

When finished, destroy the container and return the hostname to the pool:

```bash
ansible-playbook playbooks/destroy-vm.yml \
  --extra-vars provider=docker --extra-vars hostname=<hostname>
```
