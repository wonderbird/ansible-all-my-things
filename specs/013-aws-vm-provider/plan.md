# Implementation Plan: AWS EC2 Provider for Create and Destroy VM Playbooks

**Branch**: `013-aws-vm-provider` | **Date**: 2026-06-14 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/013-aws-vm-provider/spec.md`

## Summary

Add `aws` as a fourth provider for `playbooks/create-vm.yml` and
`playbooks/destroy-vm.yml`, alongside the existing `tart` (default), `docker`,
and `hcloud` providers. The new provider creates/destroys an EC2 Linux
instance via the `amazon.aws.ec2_instance` module, drawing hostnames from a new
ordered pool (`playbooks/vars/hostname_pool_aws.yml`, Dune place names),
registering/removing it in `inventories/aws_autogenerated.yml`, and applying
the exposure-based SSH host-key policy from ADR-003
(`StrictHostKeyChecking=accept-new` against a project-scoped
`inventories/aws_known_hosts`). Before launch, an idempotent
`amazon.aws.ec2_security_group` ensures the shared `ansible-sg` group exists
(SSH/22 inbound). The implementation mirrors the existing
`tart`/`docker`/`hcloud` task-file pattern (`playbooks/tasks/create/<provider>.yml`,
`playbooks/tasks/destroy/<provider>.yml`) with no new abstractions, and reuses
existing AWS defaults (`aws_default_ami_id`, `aws_default_instance_type`,
`aws_default_region`, `my_ssh_key_name`) via `include_vars` per FR-013.

## Technical Context

**Language/Version**: Ansible YAML (playbooks/tasks), consistent with the
existing `tart`/`docker`/`hcloud` provider implementations in this repo.

**Primary Dependencies**: `amazon.aws` collection (`>=7.0.0`), already declared
in `requirements.yml` and already used by `provisioners/aws-linux.yml` /
`destroy-aws-tasks.yml`.

**Storage**: Flat YAML files only —
`playbooks/vars/hostname_pool_aws.yml` (static, committed pool of 10 Dune
place names); `inventories/aws_autogenerated.yml` (generated, gitignored,
mirrors `tart_autogenerated.yml`/`docker_autogenerated.yml`/`hcloud_autogenerated.yml`
structure); `inventories/aws_known_hosts` (generated, gitignored,
project-scoped SSH known_hosts per ADR-003).

**Testing**: No Molecule scenario. The new task files are imperative
localhost orchestration against the AWS EC2 API (create/destroy a real cloud
resource), not a reusable configuration role — the same is true of the
existing `tart`/`docker`/`hcloud` create/destroy task files, none of which has
a Molecule scenario. Validation follows the spec's "Independent Test" sections:
run `create-vm.yml -e provider=aws` with AWS credentials configured against the
real AWS API, verify SSH reachability and the `inventories/aws_autogenerated.yml`
entry, then run `destroy-vm.yml -e provider=aws -e hostname=<name>` and verify
cleanup. Constitution Principle III ("Test Locally Before Cloud") does not add a
local substitute step here — there is no local equivalent of "launch an EC2
instance"; the cloud run *is* the test, exactly as scoped by the spec's
Independent Test.

**Target Platform**: Controller = `localhost` (existing `create-vm.yml`
/ `destroy-vm.yml` play target, `gather_facts: true`). Managed node = an EC2
instance running Ubuntu (per `aws_default_ami_id`), reachable over SSH from the
public internet on its assigned public IPv4, in `aws_default_region`
(`eu-north-1` by default).

**Project Type**: Single Ansible project — task-file additions to the existing
`playbooks/` tree. No new roles, no new playbooks.

**Performance Goals**: N/A (one-shot provisioning command; existing
`wait_for_connection` timeout conventions from `tart`/`docker`/`hcloud` apply).

**Constraints**: Constitution Principles I (idempotency — `ssh-keygen -R`
guards, idempotent security-group ensure, rescue cleanup), IV (YAGNI — reuse
the hcloud/docker/tart pattern verbatim), XI (DRY — `include_vars` for
`aws_default_ami_id`/`aws_default_instance_type`/`aws_default_region`/
`my_ssh_key_name` per FR-013), XII (fail loud — pool exhaustion and unknown
destroy hostname checked before any API call; AWS credential failures surface
via the AWS SDK's own error on the first API call, per FR-012), XIV (SSH
host-key verification by exposure, per ADR-003). FR-001 through FR-017 in
[spec.md](./spec.md) are the binding requirements.

**Scale/Scope**: 2 new task files (`playbooks/tasks/create/aws.yml`,
`playbooks/tasks/destroy/aws.yml`), 1 new vars file
(`playbooks/vars/hostname_pool_aws.yml`), small edits to
`playbooks/tasks/assert-provider-profile.yml` and the `assert_provider_usage_hint`
strings in `create-vm.yml`/`destroy-vm.yml`, 1 new line in `.gitignore`.
