# Contract: `destroy-vm.yml -e provider=aws -e hostname=<name>`

## Invocation

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=aws -e hostname=<name>
```

## Preconditions (checked in this order, before any AWS API call)

1. `provider` is one of `tart`, `docker`, `hcloud`, `aws` (existing
   `tasks/assert-provider-profile.yml`, extended).
2. `hostname` extra-var is defined and non-empty (existing `destroy-vm.yml`
   pre_task, unchanged).
3. `hostname` is present as a key under `all.hosts` in
   `inventories/aws_autogenerated.yml` (FR-008, FR-010). If absent, fail with a
   clear error naming the unknown hostname — before any AWS action.

## Inputs

| Variable | Required | Default | Source |
|---|---|---|---|
| `provider` | yes | — | extra-var |
| `hostname` | yes | — | extra-var |

## Effects on success — instance present in AWS

1. The EC2 instance with `instance_id` = the `aws_instance_id` recorded in the
   inventory entry is terminated
   (`amazon.aws.ec2_instance: state: absent, instance_ids: [<id>]`, FR-009).
2. Any host-key entry for that instance's `ansible_host` (read from the
   inventory entry being removed) is removed from `inventories/aws_known_hosts`
   via idempotent `ssh-keygen -f inventories/aws_known_hosts -R <ip>` (FR-009).
3. The `hostname` entry is removed from `all`, `linux`, and `aws` groups in
   `inventories/aws_autogenerated.yml` (FR-009).

## Effects on success — instance already absent/terminated in AWS (stale entry)

1. A warning is printed identifying `hostname` as stale (mirrors
   `tart`/`docker`/`hcloud` stale-entry behavior).
2. The termination API call is skipped.
3. Steps 2–3 above (known_hosts cleanup, inventory cleanup) still execute
   (FR-009) — no dangling entries remain.

## Postconditions (both paths)

- `hostname` is absent from `inventories/aws_autogenerated.yml` (asserted,
  mirrors `tart`/`docker`/`hcloud`).
- No host-key entry for the removed instance's former IP remains in
  `inventories/aws_known_hosts`.
- The shared `ansible-sg` security group is unaffected (FR-013, Assumptions).

## Out of scope / unaffected

- `provider` omitted or `tart`/`docker`/`hcloud`: behaves exactly as before
  this feature (FR-001, FR-016).
- `inventories/aws_ec2.yml`, `provisioners/aws-linux.yml`,
  `provisioners/aws-windows.yml`, `destroy.yml`, `destroy-aws-tasks.yml`, and
  the `rivendell` developer server: not read, written, or affected (FR-015).
- Windows EC2 instances: out of scope (FR-017).
