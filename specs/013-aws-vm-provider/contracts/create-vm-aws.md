# Contract: `create-vm.yml -e provider=aws`

## Invocation

```bash
ansible-playbook playbooks/create-vm.yml -e provider=aws \
  [-e aws_instance_type=<type>] [-e aws_region=<region>] [-e aws_ami_id=<ami>]
```

## Preconditions (checked in this order, before any AWS API call)

1. `provider` is one of `tart`, `docker`, `hcloud`, `aws` (existing
   `tasks/assert-provider.yml`, extended).
2. At least one entry in `playbooks/vars/hostname_pool_aws.yml`
   (`hostname_pool`) is not already a key under `all.hosts` in
   `inventories/aws_autogenerated.yml` (FR-007). Failure message names
   `playbooks/vars/hostname_pool_aws.yml`.

AWS credential validity is NOT pre-checked (FR-012) — an invalid/missing
credential chain surfaces as an AWS SDK authentication error on the first
`amazon.aws` module call (the security-group ensure step), after the hostname
has been selected.

## Inputs

| Variable | Required | Default | Source |
|---|---|---|---|
| `provider` | yes | — | extra-var |
| `aws_instance_type` | no | `aws_default_instance_type` | `inventories/group_vars/aws_ec2_linux/vars.yml` |
| `aws_ami_id` | no | `aws_default_ami_id` | `inventories/group_vars/aws_ec2_linux/vars.yml` |
| `aws_region` | no | `aws_default_region` | `inventories/group_vars/aws_ec2/vars.yml` |
| `my_ssh_key_name` | no (vault-provided) | — | `inventories/group_vars/all/vars.yml` (vault) |

## Effects on success

1. The shared `ansible-sg` security group exists in `aws_region` (ensured
   idempotently, FR-013), allowing inbound TCP/22 from `0.0.0.0/0`.
2. Exactly one new EC2 instance is created (FR-002) with `Name` tag = the next
   available AWS hostname, the resolved `image_id`, `instance_type`, `region`,
   `key_name: my_ssh_key_name`, and `security_groups: [ansible-sg]` (FR-013).
3. `inventories/aws_known_hosts` has any stale entry for the new instance's
   public IP removed (idempotent `ssh-keygen -R`, FR-005).
4. `wait_for_connection` succeeds against the new instance's public IPv4 on
   port 22 as `ubuntu`, with
   `ansible_ssh_common_args: "-o StrictHostKeyChecking=accept-new -o UserKnownHostsFile=inventories/aws_known_hosts"`.
5. `inventories/aws_autogenerated.yml` gains one entry under `all`, `linux`,
   and `aws` groups for the new hostname, with `ansible_host` (public IPv4),
   `ansible_port: 22`, `ansible_user: ubuntu`, `aws_instance_id` (EC2
   instance-id), and the `ansible_ssh_common_args` from step 4 (FR-004,
   FR-005).
6. The assigned hostname is printed (FR-006).

## Effects on failure (after instance creation, before completion)

- The partially-created EC2 instance is terminated
  (`amazon.aws.ec2_instance: state: absent, instance_ids: [<id>]`) before the
  playbook fails (FR-014).
- `inventories/aws_autogenerated.yml` and `inventories/aws_known_hosts` are
  left unchanged (no entry was written yet).
- The `ansible-sg` security group, if it was created/already existed, is left
  in place (shared resource, not part of rollback).

## Out of scope / unaffected

- `provider` omitted or `tart`/`docker`/`hcloud`: behaves exactly as before
  this feature (FR-001, FR-016).
- `inventories/aws_ec2.yml` (dynamic inventory plugin),
  `provisioners/aws-linux.yml`, `provisioners/aws-windows.yml`, `destroy.yml`,
  `destroy-aws-tasks.yml`, and the `rivendell` developer server: not read,
  written, or affected (FR-015).
- Windows EC2 instances: out of scope (FR-017).
