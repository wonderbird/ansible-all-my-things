# Quickstart: Configure Basic Profile for Tart VMs

## Prerequisites

- macOS ARM64 host (control node)
- Tart CLI installed (`brew install tart`)
- `sshpass` installed (`brew install sshpass`)
- `my_ansible_user_password` and `my_ssh_public_key` configured in vault
  (`inventories/group_vars/all/vault.yml`)

## Two-step operator workflow

### Step 1: Create the VM

```bash
ansible-playbook playbooks/create-vm.yml
```

Prints the assigned hostname on completion. The VM is registered in
`inventories/tart_autogenerated.yml` under the `all`, `linux`, and `tart`
groups, reachable via the default `admin` bootstrap account.

### Step 2: Configure the basic profile

```bash
ansible-playbook playbooks/configure-profile.yml
```

No extra-vars required. On completion:

- `my_ansible_user` and all `desktop_users` exist with sudo and SSH key
  access (password login disabled)
- The package cache is current, all packages are upgraded, timezone is
  `Europe/Berlin`
- Each desktop user has NVM, the current Node.js LTS, and the global npm
  tools `eslint`, `markdownlint-cli`, `prettier`, `typescript`
- podman, Ruby, Python, the Dolt SQL server, and the Claude Code CLI are
  installed and configured
- The VM reboots and comes back online if the OS requires it

## Verify

This section is a reproducible, scenario-by-scenario acceptance test
procedure matching `spec.md`'s Acceptance Scenarios, written so an agent with
shell access to the macOS ARM64 + tart host can run it end-to-end and
self-report PASS/FAIL — no human needs to relay command output. The only
human-required steps are the one-time host **Prerequisites** above (tart,
sshpass, vault secrets); everything below is agent-executable.

Run the **Two-step operator workflow** above first (`create-vm.yml` then
`configure-profile.yml`), capturing the hostname `create-vm.yml`
prints. Then run the following as a single shell session:

```bash
HOSTNAME=<hostname-from-create-vm-output>
export VM_HOST=$(ansible-inventory -i inventories/tart_autogenerated.yml --host "$HOSTNAME" | jq -r '.ansible_host')
ADMIN_USER=gandalf      # my_ansible_user
DESKTOP_USER=galadriel  # first entry in desktop_users

# If VM_HOST's IP was previously used by a different VM, clear the stale host key
ssh-keygen -R "$VM_HOST" >/dev/null 2>&1

ssh_run() { ssh -o BatchMode=yes -o StrictHostKeyChecking=accept-new "$1@$VM_HOST" "${@:2}"; }
# Sources NVM directly (bypasses ~/.bashrc's non-interactive-shell guard)
nvm_env='export NVM_DIR="$HOME/.nvm"; . "$NVM_DIR/nvm.sh"; export PATH="$HOME/.local/bin:$PATH"'

# Scenario 1: sudo membership + key-based SSH
g_admin=$(ssh_run "$ADMIN_USER" "groups")
g_desktop=$(ssh_run "$DESKTOP_USER" "groups")
sudo_ok=$(ssh_run "$ADMIN_USER" "sudo -n true && echo SUDO_OK")
if grep -q sudo <<<"$g_admin" && grep -q sudo <<<"$g_desktop" && [ "$sudo_ok" = SUDO_OK ]; then
  echo "Scenario 1: PASS"
else
  echo "Scenario 1: FAIL - admin groups=[$g_admin] desktop groups=[$g_desktop] sudo_ok=[$sudo_ok]"
fi

# Scenario 2: SSH password authentication disabled (effective config, not just the file)
pwauth=$(ssh_run "$ADMIN_USER" "sudo sshd -T | grep -i passwordauthentication")
if [ "$pwauth" = "passwordauthentication no" ]; then
  echo "Scenario 2: PASS"
else
  echo "Scenario 2: FAIL - sshd -T reports [$pwauth] (Include drop-ins under /etc/ssh/sshd_config.d/ may override sshd_config; see ansible-all-my-things-85t5)"
fi

# Scenario 3: package cache, upgrades, timezone
tz=$(ssh_run "$ADMIN_USER" "timedatectl show -p Timezone --value")
upgradable=$(ssh_run "$ADMIN_USER" "apt list --upgradable 2>/dev/null | grep -v '^Listing'")
deferred=$(ssh_run "$ADMIN_USER" "sudo apt-get -s dist-upgrade 2>&1 | grep -E '^(The following upgrades have been deferred|.*kept back:)'")
if [ "$tz" = "Europe/Berlin" ] && { [ -z "$upgradable" ] || [ -n "$deferred" ]; }; then
  echo "Scenario 3: PASS"
else
  echo "Scenario 3: FAIL - timezone=[$tz] upgradable=[$upgradable] deferred=[$deferred]"
fi

# Scenario 4: Node.js toolchain for desktop users
nvm_node=$(ssh_run "$DESKTOP_USER" "$nvm_env; command -v nvm && node --version")
npm_globals=$(ssh_run "$DESKTOP_USER" "$nvm_env; npm ls -g --depth=0")
missing=""
for pkg in eslint markdownlint-cli prettier typescript; do
  grep -q "$pkg" <<<"$npm_globals" || missing="$missing $pkg"
done
if [ -n "$nvm_node" ] && [ -z "$missing" ]; then
  echo "Scenario 4: PASS"
else
  echo "Scenario 4: FAIL - nvm/node=[$nvm_node] missing global npm packages:[$missing]"
fi

# Scenario 5: development tool roles
versions=$(ssh_run "$ADMIN_USER" "podman --version && ruby --version && python3 --version && dolt version")
dolt_active=$(ssh_run "$ADMIN_USER" "sudo systemctl is-active dolt-sql-server")
claude_version=$(ssh_run "$DESKTOP_USER" "$nvm_env; claude --version")
if [ -n "$versions" ] && [ "$dolt_active" = "active" ] && [ -n "$claude_version" ]; then
  echo "Scenario 5: PASS"
else
  echo "Scenario 5: FAIL - versions=[$versions] dolt_active=[$dolt_active] claude=[$claude_version]"
fi

# Scenario 6: reboot-if-required
reboot_pending=$(ssh_run "$ADMIN_USER" "ls /var/run/reboot-required* 2>/dev/null")
if [ -z "$reboot_pending" ]; then
  echo "Scenario 6: PASS (no pending reboot; the preceding configure-profile.yml run must also have completed with failed=0)"
else
  echo "Scenario 6: FAIL - reboot still pending: [$reboot_pending]"
fi
```

## Re-run check (US2 idempotency)

```bash
ansible-playbook playbooks/configure-profile.yml --limit "$HOSTNAME" -v 2>&1 \
  | sed -E 's/\x1b\[[0-9;]*m//g' > /tmp/configure-profile-rerun.log

if grep -qE "^$HOSTNAME +: ok=[0-9]+ +changed=0 " /tmp/configure-profile-rerun.log; then
  echo "US2: PASS"
else
  echo "US2: FAIL - changed tasks present, see below"
  grep -B8 "changed: \[$HOSTNAME\]" /tmp/configure-profile-rerun.log
fi
```

`no_log: true` tasks censor their item details in the diagnostic grep above;
the preceding `TASK [...]` line is still printed even for `no_log` tasks —
cross-reference it against `playbooks/setup-users.yml` /
`playbooks/setup-nodejs.yml` to identify which task is non-idempotent.
