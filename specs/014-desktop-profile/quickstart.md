# Quickstart: Desktop Profile for create-vm.yml / configure-profile.yml

## Prerequisites

- Same per-provider prerequisites as today (tart: macOS host; hcloud:
  `HCLOUD_TOKEN`; aws: AWS credential chain + `my_ssh_key_name`).
- `docker` is NOT usable with `profile=desktop` (rejected loudly) — use
  `tart`, `hcloud`, or `aws`.
- Vault password available via `ANSIBLE_VAULT_PASSWORD_FILE` (existing
  requirement for `configure-profile.yml`'s desktop-only steps, same as the
  legacy `configure-linux.yml` path).

## Create a desktop-profile VM

```bash
ansible-playbook playbooks/create-vm.yml -e provider=tart -e profile=desktop
```

- Picks the next unused hostname from the `tart` pool (unchanged mechanics).
- Registers the new host under `all`, `linux`, `tart` (unchanged) **and**
  under `desktop` (new) in `inventories/tart_autogenerated.yml`.
- Prints `VM created with hostname: <name>` (unchanged).

Configure it:

```bash
ansible-playbook configure-profile.yml -i inventories/tart_autogenerated.yml --limit <name>
```

- Applies the same shared steps as the `basic` profile
  (`setup-users.yml`/`setup-basics.yml`/`setup-nodejs.yml`).
- Additionally imports `setup-desktop.yml`, `setup-keyring.yml`,
  `setup-desktop-apps.yml` verbatim (XFCE + XRDP desktop, GNOME keyring,
  Chromium/VS Code/git setup).
- Applies the full legacy role list (`podman`, `ruby`, `python`,
  `claude_code`, `tmux`, `nerd_font`, `dolt_sql_server`, `google_chrome`)
  instead of the `basic` profile's 6-role subset.

## Create a basic-profile VM (unchanged shape, now with an explicit group)

```bash
ansible-playbook playbooks/create-vm.yml -e provider=tart -e profile=basic
ansible-playbook playbooks/create-vm.yml -e provider=tart                   # profile=basic is the default
```

- Behaves exactly as before this feature, except the new host now also
  lands in the `basic` inventory group (previously had no profile group at
  all).

## AWS: RDP access scoped to desktop VMs only

```bash
ansible-playbook playbooks/create-vm.yml -e provider=aws -e profile=desktop
```

- The shared `ansible-sg` security group gains an inbound TCP/3389 rule
  (in addition to the existing TCP/22 rule).
- A `provider=aws -e profile=basic` (or default) create run does NOT add
  this rule — basic-profile AWS VMs stay SSH-only.

## Rejected combination

```bash
ansible-playbook playbooks/create-vm.yml -e provider=docker -e profile=desktop
```

```text
FAILED! => "Unsupported combination: provider=docker is incompatible with
profile=desktop (the minimized docker image has no desktop support).
Use provider=tart, hcloud, or aws for a desktop VM."
```

Fails immediately — no container created.

## Destroy (unchanged invocation)

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=tart -e hostname=<name>
```

- Removes the host from `all`/`linux`/`<provider>` (unchanged) and from
  whichever of `basic`/`desktop` it was registered under at create time —
  no `profile` extra-var needed on destroy.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `Unsupported combination: provider=docker is incompatible with profile=desktop...` | Requested a desktop VM on the `docker` provider | Use `tart`, `hcloud`, or `aws` |
| `Unsupported profile '<x>'...` | Typo or unsupported `profile` value | Use `basic` or `desktop` |
| Desktop VM has no RDP access (AWS) | VM was created with `profile=basic` or before this feature shipped | Re-create with `profile=desktop`, or manually add the 3389 rule to `ansible-sg` for that one-off case |
| Configuring a `desktop`-group host installs only the `basic` role set | Targeted the wrong `--limit`/inventory group, or the host was actually created with `profile=basic` | Check `inventories/<provider>_autogenerated.yml` for the host's group membership |
