# Contract: `create-vm.yml -e profile=<basic|desktop>`

## Invocation

```bash
ansible-playbook playbooks/create-vm.yml -e provider=<tart|hcloud|aws> -e profile=desktop
ansible-playbook playbooks/create-vm.yml [-e provider=<tart|docker|hcloud|aws>] [-e profile=basic]
```

## Preconditions (checked in this order, before any infrastructure action)

1. `provider` is one of `tart`, `docker`, `hcloud`, `aws` (existing
   `tasks/assert-provider-profile.yml`, unchanged).
2. `profile` is one of `basic`, `desktop` (FR-001; new assertion).
3. The combination `provider=docker` with `profile=desktop` is rejected with
   an explicit error naming both values (FR-006; new assertion).
4. The selected provider's hostname-pool exhaustion check (existing,
   unchanged per provider).

## Inputs

| Variable | Required | Default | Source |
|---|---|---|---|
| `provider` | no | `tart` | extra-var (unchanged) |
| `profile` | no | `basic` | extra-var (new) |

## Effects on success

1. Exactly one new VM/instance/container is created via the selected
   provider's existing create task file (unchanged behavior for the
   create/destroy mechanics themselves).
2. The new host is registered in
   `inventories/<provider>_autogenerated.yml` under `all`, `linux`,
   `<provider>` (unchanged), **and** under `basic` or `desktop` (new),
   matching the `profile` value used.
3. When `provider=aws` and `profile=desktop`: the shared `ansible-sg`
   security group additionally allows inbound TCP 3389 (FR-007). When
   `provider=aws` and `profile=basic`: no 3389 rule is added (FR-008).
4. The assigned hostname is printed (unchanged).

## Effects on failure

- `profile` not in `{basic, desktop}`: fails immediately, no infrastructure
  action, no hostname consumed.
- `provider=docker profile=desktop`: fails immediately, no container
  created, no hostname consumed, error names both `provider` and `profile`.
- All other failure modes (pool exhaustion, provider-specific rollback):
  unchanged from the existing per-provider behavior.

## Out of scope / unaffected

- `profile` omitted: behaves exactly as `profile=basic` (FR-001 default).
- `destroy-vm.yml` takes no `profile` extra-var — destroy removes a host
  from whichever profile group it was registered under at create time,
  inferred from the existing inventory entry, not re-specified by the
  caller.
