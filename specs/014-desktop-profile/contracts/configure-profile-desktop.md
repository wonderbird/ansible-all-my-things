# Contract: `configure-profile.yml` against a `desktop`-group host

## Invocation

```bash
ansible-playbook configure-profile.yml -i inventories/<provider>_autogenerated.yml --limit desktop
```

(Same invocation shape as the existing `basic` profile; the inventory group
membership set at create time, not an extra-var, selects which branch
applies to a given host.)

## Preconditions

- The target host is a member of the `desktop` inventory group (set by
  `create-vm.yml -e profile=desktop` at create time; FR-002).
- The target host is reachable over SSH (existing precondition, unchanged).

## Effects on success

1. `setup-users.yml`, `setup-basics.yml`, `setup-nodejs.yml` run exactly as
   they do for the `basic` profile (FR-012 — unchanged for shared steps).
2. `setup-desktop.yml`, `setup-keyring.yml`, `setup-desktop-apps.yml` run
   verbatim, unmodified, against the host (FR-005) — installing the desktop
   environment (XFCE + XRDP), GNOME keyring, and desktop applications
   (Chromium, VS Code, git config) exactly as they do today for legacy
   `configure-linux.yml` targets.
3. `configure-profile-roles.yml`'s `hosts: desktop` play applies the legacy
   `configure-linux-roles.yml` role list verbatim (FR-004): `podman`,
   `ruby`, `python`, `claude_code`, `tmux`, `nerd_font`, `dolt_sql_server`,
   `google_chrome` (tagged `not-supported-on-vagrant-arm64`).
4. `reboot-if-required.yml` runs exactly as for the `basic` profile.

## Effects on failure

- Same per-step failure behavior as the existing `basic`-profile run and the
  legacy `configure-linux.yml` run of the same imported playbooks/roles — no
  new failure handling introduced (the playbooks/roles are imported
  unmodified).

## Out of scope / unaffected

- `restore.yml`, `setup-homebrew.yml`: not imported by this branch (FR-009,
  FR-010).
- A host in the `basic` group: `configure-profile.yml`'s existing branch is
  unchanged (FR-012); it never runs the desktop-only playbooks above.
- Role extraction of `setup-desktop.yml`/`setup-keyring.yml`/
  `setup-desktop-apps.yml`/the legacy role list into standalone roles: out of
  scope, logged as an accepted exception in plan.md's Complexity Tracking.
