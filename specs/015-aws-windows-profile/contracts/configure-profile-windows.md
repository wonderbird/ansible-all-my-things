# Contract: `configure-profile.yml` against a `windows`-group host

## Invocation

```bash
ansible-playbook -i inventories/aws_autogenerated.yml --limit <windows-host> playbooks/configure-profile.yml
```

Identical command shape already used to configure any host — no new
extra-var, no new inventory file argument.

## Preconditions

- The target host is a member of the `windows` inventory group (set by
  `create-vm.yml -e profile=windows`; see `create-vm-profile-windows.md`).
- Ansible Vault password available via `ANSIBLE_VAULT_PASSWORD_FILE`
  (existing requirement, unchanged).

## Effects on success

1. `setup-users.yml`, `setup-basics.yml`, and `playbooks/reboot-if-required.yml`
   — each now scoped to `hosts: linux` — do **not** run against the host
   (FR-011), since a `windows`-group host is never a `linux`-group member.
2. `configure-profile-roles.yml`'s `basic`/`desktop` plays do not run against
   the host, for the same reason (no change needed to that file — it is
   already scoped to `basic`/`desktop`, neither of which a `windows` host
   joins).
3. `setup-roles-windows.yml` (`hosts: windows`) runs, applying the
   `windows_foundation` role (RDP performance tuning, Chocolatey, a pending
   reboot if required) followed by the `win_ai_agent` role (Claude Code +
   nvm + Node.js LTS via Chocolatey, a pending reboot if required) — both
   verbatim, unmodified (FR-010, FR-013).

## Effects on `basic`/`desktop` hosts (regression check)

- Every existing target of `setup-users.yml`/`setup-basics.yml`/
  `reboot-if-required.yml` is already a `linux`-group member (Assumptions in
  spec.md), so re-scoping these three plays from `hosts: all` to
  `hosts: linux` is behavior-preserving: identical hosts targeted, identical
  tasks run, in identical order (FR-012).

## Out of scope / unaffected

- No change to `configure-profile-roles.yml` (already correctly scoped).
- No change to `windows_foundation`/`win_ai_agent` role internals (FR-013).
- No change to the legacy `configure-windows.yml`/`setup-roles-windows.yml`
  entrypoints themselves — `configure-profile.yml` imports
  `setup-roles-windows.yml` by reference, it is not copied or rewritten.
