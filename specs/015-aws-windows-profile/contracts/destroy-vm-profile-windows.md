# Contract: `destroy-vm.yml -e provider=aws -e hostname=<windows-host>`

## Invocation

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=aws -e hostname=<name>
```

Identical command shape already used to destroy any `provider=aws` VM — no
new extra-var. The host's profile (and therefore whether it is a `windows`
host) is inferred from its existing inventory entry, not re-specified by the
caller.

## Preconditions (checked in this order, before any infrastructure action)

1. `provider` is one of `tart`, `docker`, `hcloud`, `aws` (existing,
   unchanged).
2. `hostname` is defined and non-empty (existing, unchanged).
3. `hostname` is present in `inventories/aws_autogenerated.yml` (existing,
   unchanged) — failing loudly with "nothing to destroy" if absent.

## Effects on success

1. The EC2 instance is terminated (existing mechanism, unchanged —
   instance ID and IP are read from the existing inventory entry regardless
   of whether the host is `linux` or `windows`).
2. The instance's known_hosts entry is removed (existing, unchanged).
3. `<name>` is removed from every group it belongs to in
   `inventories/aws_autogenerated.yml`: `all`, `aws`, and — new — `windows`
   (alongside the existing `basic`/`desktop` cleanup, none of which apply to
   a `windows` host since it was never added to them).

## Effects on failure

- `hostname` absent from inventory: fails immediately, no AWS action
  (existing, unchanged).
- EC2 instance already terminated/absent from AWS (stale inventory entry):
  warns and proceeds with inventory cleanup only (existing, unchanged).

## Out of scope / unaffected

- Destroying a `basic`/`desktop`/Linux host: unchanged (FR-009 — this
  feature does not alter cleanup for any other host).
- No `profile` extra-var is introduced on `destroy-vm.yml` (consistent with
  `014-desktop-profile`'s precedent).
