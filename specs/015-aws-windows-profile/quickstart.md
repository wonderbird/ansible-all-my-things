# Quickstart: AWS Windows Profile for create-vm.yml / configure-profile.yml

## Prerequisites

- AWS credential chain configured, `my_ssh_key_name`/`my_ssh_public_key` set
  (same prerequisites as any other `provider=aws` VM).
- `windows_admin_password` set in Vault (`vault_windows_admin_password` in
  `inventories/group_vars/all/vault.yml`) — already a documented requirement,
  not new.
- `profile=windows` is usable only with `provider=aws` — use `tart`,
  `docker`, or `hcloud` for Linux VMs.
- Vault password available via `ANSIBLE_VAULT_PASSWORD_FILE` (existing
  requirement for `configure-profile.yml`, unchanged).

## Create an aws/windows VM

```bash
ansible-playbook playbooks/create-vm.yml -e provider=aws -e profile=windows
```

- Picks the next unused hostname from the existing, shared AWS pool
  (`playbooks/vars/hostname_pool_aws.yml` — unchanged mechanics, no separate
  Windows pool).
- Launches a Windows Server EC2 instance using the AMI/instance-type already
  defaulted in `inventories/group_vars/aws_ec2_windows/vars.yml`.
- Registers the new host under `all`, `aws`, and `windows` (new) in
  `inventories/aws_autogenerated.yml` — **not** under `linux`.
- Opens inbound TCP 3389 (RDP) on the shared `ansible-sg` security group, in
  addition to the existing TCP 22 (SSH) rule.
- Prints `VM created with hostname: <name>` (unchanged).

## Configure it

```bash
ansible-playbook -i inventories/aws_autogenerated.yml --limit <name> playbooks/configure-profile.yml
```

- Applies the `windows_foundation` role (RDP tuning, Chocolatey, reboot if
  pending) and the `win_ai_agent` role (Claude Code, nvm, Node.js LTS,
  reboot if pending) — the same role set already proven by the legacy
  `setup-roles-windows.yml`.
- Does **not** apply `setup-users.yml`/`setup-basics.yml`/
  `reboot-if-required.yml` (Linux-only; re-scoped to `hosts: linux`, which a
  `windows`-group host is never a member of) or the `basic`/`desktop` role
  lists in `configure-profile-roles.yml`.

## Connect

```bash
ssh -i <your-key> Administrator@<public-ip>
```

- PowerShell shell (configured via
  `inventories/group_vars/windows/vars.yml`'s `ansible_shell_type:
  powershell`), authenticated with the same SSH key already used for Linux
  AWS VMs.
- RDP also available on port 3389 using the `windows_admin_password` Vault
  secret.

## Rejected combination

```bash
ansible-playbook playbooks/create-vm.yml -e provider=tart -e profile=windows
```

```text
FAILED! => "Unsupported combination: provider=tart is incompatible with
profile=windows (Windows is only supported on provider=aws)."
```

Fails immediately — no VM created, no hostname consumed.

## Destroy (unchanged invocation)

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=aws -e hostname=<name>
```

- Terminates the EC2 instance, removes its known_hosts entry, and removes it
  from `all`, `aws`, and `windows` in `inventories/aws_autogenerated.yml` —
  no `profile` extra-var needed on destroy.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `Unsupported combination: provider=<x> is incompatible with profile=windows...` | Requested a Windows VM on a non-`aws` provider | Use `provider=aws` |
| `windows_admin_password must be set...` | Vault secret missing | Set `vault_windows_admin_password` in `inventories/group_vars/all/vault.yml` |
| `Unsupported profile '<x>'...` | Typo or unsupported `profile` value | Use `basic`, `desktop`, or `windows` |
| `configure-profile.yml` fails with an `apt`/`user` module error against a Windows host | Targeted a `windows`-group host with a stale checkout that still scopes `setup-users.yml`/`setup-basics.yml` to `hosts: all` | Ensure this feature's re-scoping (`hosts: linux`) has landed |
| SSH connection refused right after `create-vm.yml` completes | Windows boots slower than Linux; OpenSSH service start lags instance readiness | Wait the timeout already built into `wait_for_connection`, then retry |
