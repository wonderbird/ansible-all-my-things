# Data Model: Docker Provider for Create and Destroy VM Playbooks

## Entities

### Provider Selector

Top-level extra-var on `create-vm.yml` and `destroy-vm.yml` that dispatches
between providers.

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| `provider` | `str` | `-e provider=<value>` | Allowed values: `tart` (default), `docker` |

**Validation rules**:

- MUST be one of `tart` / `docker`; any other value MUST `fail` before any
  task dispatch (Principle XII)
- Default (`tart`) MUST preserve pre-existing behavior exactly (FR-001,
  FR-017)

---

### Docker Hostname Pool

Ordered list of available container hostnames, independent from and disjoint
with the Tart hostname pool (`playbooks/vars/hostname_pool_tart.yml`).

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| `hostname_pool` | `list[str]` | `playbooks/vars/hostname_pool_docker.yml` | Same variable name as the Tart pool; loaded conditionally so only one is in scope per invocation (FR-003) |

**Initial values**: `tatooine`, `coruscant`, `naboo`, `alderaan`, `hoth`,
`dagobah`, `endor`, `kamino`, `mustafar`, `yavin` (FR-011)

**Validation rules**:

- List MUST be non-empty at playbook start (Principle XII)
- All names MUST be lowercase strings (Docker container/hostname
  constraints; consistency with Tart pool)
- MUST NOT collide with `hostname_pool_tart.yml` (Tart pool) — disjoint by
  construction (Star Wars vs. Star Trek naming)

**State**: Append-only, identical semantics to the Tart pool — names are
allocated (in-use) or available based on
`inventories/docker_autogenerated.yml` membership; never removed from the
pool file itself.

---

### Docker Inventory Entry

Per-container record in `inventories/docker_autogenerated.yml`. A container
is "known" if and only if its hostname appears in `all.hosts`.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `ansible_host` | `str` | Yes | Always `127.0.0.1` (FR-005) |
| `ansible_port` | `int` | Yes | Host port dynamically published for container port 22, discovered via `docker port <hostname> 22/tcp` |
| `ansible_user` | `str` | Yes | Always `root` (FR-005) |
| `ansible_ssh_pass` | `str` | Yes | From `docker_root_password` in `playbooks/vars/docker_credentials.yml` |
| `ansible_ssh_common_args` | `str` | Yes | Disables host-key checking (each fresh container has a new SSH host key) |

**Group membership** (all three groups updated on create/destroy):

| Group | Purpose |
|-------|---------|
| `all` | Root group; carries connection vars |
| `linux` | Selects Linux-targeted roles |
| `docker` | Selects Docker-specific vars |

**Validation rules**:

- `destroy-vm.yml -e provider=docker` MUST fail before any Docker action if
  `hostname` is absent from `all.hosts` (FR-010, Principle XII)
- `create-vm.yml -e provider=docker` MUST NOT create an entry for a hostname
  already in `all.hosts` (idempotency guard — same as Tart provider; in
  practice unreachable since hostnames are selected from the
  pool-minus-used set)
- `ansible_port` MUST be validated as a non-empty value parsed from
  `docker port` output before being written to inventory (Principle XII)

**State transitions**:

```text
[absent]  →  create-vm.yml -e provider=docker  →  [present in all + linux + docker]
[present] →  destroy-vm.yml -e provider=docker →  [absent from all + linux + docker]
```

### Docker Container Config

Ansible variables controlling container resource allocation. Defined as
defaults; overridable via `--extra-vars`.

| Variable | Default | Type | Notes |
|----------|---------|------|-------|
| `docker_vm_cpus` | `2` | `int` | `--cpus` value (FR-012) |
| `docker_vm_memory` | `4g` | `str` | `--memory` value (FR-012) |

**Source**: defined at the top of `create-vm.yml`, parallel to `vm_cpus` /
`vm_memory_mb` for the Tart provider. Disk size limits are not applicable
(FR-012) and have no corresponding variable.

---

### Container Image

The reusable, idempotently-built image artefact.

| Field | Value | Notes |
|-------|-------|-------|
| Tag | `ansible-vm-docker:latest` | Fixed name (FR-013) |
| Source | `playbooks/files/docker/Dockerfile` | `ubuntu:24.04` base, systemd entrypoint, sshd enabled |
| Build trigger | `docker image inspect ansible-vm-docker:latest` returns non-zero | Build-once, no auto-rebuild (FR-013, clarified 2026-06-11) |
| Build arg | `DOCKER_ROOT_PASSWORD` | Sourced from `docker_root_password` (`playbooks/vars/docker_credentials.yml`); baked into the image via `chpasswd` |

**State**: Build-once-reuse-forever for the lifetime of the local image
cache. Rotating the root password requires manually removing the
`ansible-vm-docker:latest` tag to force a rebuild (out of scope for this
feature — no rotation mechanism is implemented, YAGNI).

## Relationships

```text
HostnamePool (hostname_pool_docker.yml)
    │  first unused name (vs. docker_autogenerated.yml all.hosts)
    ▼
hostname (str)
    │
    ├──▶ Container Image (build-once-guard) ──▶ ansible-vm-docker:latest
    │
    ├──▶ docker run --name <hostname> --hostname <hostname> ...
    │         │
    │         └──▶ docker port <hostname> 22/tcp ──▶ ansible_port
    │
    └──▶ Docker Inventory Entry ──written──▶ inventories/docker_autogenerated.yml
```
