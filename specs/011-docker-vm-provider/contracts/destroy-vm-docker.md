# Contract: destroy-vm.yml -e provider=docker

## Invocation

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=docker -e hostname=tatooine
```

## Parameters

| Name | Required | Default | Description |
|------|----------|---------|-------------|
| `provider` | Yes (for this path) | `tart` | Set to `docker` to select this provider |
| `hostname` | Yes | — | Name of the container to destroy; must match an entry in `inventories/docker_autogenerated.yml` |

## Preconditions

- `hostname` is present in `inventories/docker_autogenerated.yml` under
  `all.hosts` (otherwise fails immediately before any action, FR-010)
- `docker` CLI and a running Docker daemon are available on the control node
  (required for the `docker container inspect` existence check and, if the
  container exists, `docker stop`/`docker rm`)

## Postconditions (on success)

- If the container exists in Docker: it is stopped and removed
  (`docker stop` + `docker rm`)
- If the container does NOT exist in Docker (stale entry): a warning is
  printed and the stop/remove steps are skipped gracefully (FR-009)
- In both cases, all references to `hostname` are removed from
  `inventories/docker_autogenerated.yml` (groups `all`, `linux`, `docker`)

## Failure modes

| Condition | Behaviour |
|-----------|-----------|
| `provider` value not `tart`/`docker` | Fails immediately with an actionable error before any task dispatch |
| `hostname` not in `inventories/docker_autogenerated.yml` | Fails immediately with actionable error identifying the unknown hostname; no Docker action taken (FR-010) |
| `hostname` not provided | Ansible variable undefined error at task evaluation (mirrors Tart contract) |
| Container present in inventory but absent from Docker | Logs a warning; removes the stale inventory entry; does not fail (FR-009) |

## Idempotency

Partially idempotent: running destroy twice for the same hostname fails on
the second run (hostname no longer in `inventories/docker_autogenerated.yml`)
rather than silently succeeding. This is intentional — a missing hostname is
always an error (FR-010), exactly as for the Tart provider.

## Out of scope (unaffected by this contract)

- `inventories/vagrant_docker.yml` and `test/docker/*` are not read, written,
  or removed (FR-016).
- Omitting `provider` (or passing `provider=tart`) is unaffected — see
  `destroy-vm.md` (Tart contract, unchanged, FR-017).
