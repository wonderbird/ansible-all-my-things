# Contract: create-vm.yml -e provider=docker

## Invocation

```bash
ansible-playbook playbooks/create-vm.yml -e provider=docker
ansible-playbook playbooks/create-vm.yml -e provider=docker -e docker_vm_cpus=4 -e docker_vm_memory=8g
```

## Parameters

| Name | Required | Default | Description |
|------|----------|---------|-------------|
| `provider` | Yes (for this path) | `tart` | Set to `docker` to select this provider |
| `docker_vm_cpus` | No | `2` | `--cpus` value passed to `docker run` |
| `docker_vm_memory` | No | `4g` | `--memory` value passed to `docker run` |

No extra-vars beyond `provider=docker` are required for the common case
(SC-001).

## Preconditions

- At least one hostname in `playbooks/vars/hostname_pool_docker.yml` is not
  present in `inventories/docker_autogenerated.yml` (otherwise fails before
  any container or image action)
- `docker` CLI and a running Docker daemon are available on the control node
- `sshpass` is installed on the control node (already required by the Tart
  provider)

## Postconditions (on success)

- The image `ansible-vm-docker:latest` exists (built if it did not already)
- A new Docker container is running, named and hostnamed to the assigned
  pool entry, with SSH reachable on `127.0.0.1:<dynamic-port>`
- `inventories/docker_autogenerated.yml` contains an entry for the new
  container in groups `all`, `linux`, and `docker`
- The assigned hostname is printed as a debug message on completion (FR-006)

## Failure modes

| Condition | Behaviour |
|-----------|-----------|
| `provider` value not `tart`/`docker` | Fails immediately with an actionable error before any task dispatch |
| Docker hostname pool exhausted | Fails immediately with error naming `playbooks/vars/hostname_pool_docker.yml`; no image build or container action taken (FR-007) |
| `docker` CLI/daemon unavailable | Command task fails explicitly; no inventory entry written |
| Image build fails | Task fails with error; no container is started, no inventory entry written |
| `docker port` returns no/invalid port | Task fails with error; inventory write does not proceed |
| SSH readiness poll times out | Rescue block runs (see below); playbook fails |
| Failure after container start | Rescue block runs: `docker stop` + `docker rm` (both `failed_when: false`); fail-loud message naming the hostname (FR-015) |

## Idempotency

- Image build step is idempotent: `ansible-vm-docker:latest` is built only if
  absent; never rebuilt automatically (FR-013).
- Container creation is NOT idempotent by design — each invocation creates
  one new container with the next pool hostname (FR-002), exactly as the
  Tart provider creates one new VM per invocation (FR-001 baseline). A
  hostname already present in `inventories/docker_autogenerated.yml` is
  never re-selected (pool-difference check).

## Out of scope (unaffected by this contract)

- `inventories/vagrant_docker.yml` and `test/docker/*` are not read, written,
  or removed (FR-016).
- Omitting `provider` (or passing `provider=tart`) is unaffected — see
  `create-vm.md` (Tart contract, unchanged, FR-017).
