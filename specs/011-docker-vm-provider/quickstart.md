# Quickstart: Docker Provider for Create and Destroy VM Playbooks

## Prerequisites

- Any host with a working Docker daemon (Linux primary; any host that is not
  macOS ARM64 also works for the Docker provider)
- `sshpass` installed (already required by the Tart provider)

## Create a Docker container

```bash
ansible-playbook playbooks/create-vm.yml -e provider=docker
```

On first run, this builds the `ansible-vm-docker:latest` image (subsequent
runs reuse it). Prints the assigned hostname on completion. The container is
immediately reachable via the inventory.

Override resources:

```bash
ansible-playbook playbooks/create-vm.yml -e provider=docker -e docker_vm_cpus=4 -e docker_vm_memory=8g
```

## Destroy a Docker container

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=docker -e hostname=tatooine
```

## Check inventory

```bash
cat inventories/docker_autogenerated.yml
```

## SSH into a container

```bash
ansible tatooine -m ping
sshpass -p <docker_root_password> ssh -o StrictHostKeyChecking=no -p <ansible_port> root@127.0.0.1
```

(`<docker_root_password>` is the value in `playbooks/vars/docker_credentials.yml`;
`<ansible_port>` is the dynamically published SSH port recorded in
`inventories/docker_autogenerated.yml`.)

## Pool management

Available hostnames live in `playbooks/vars/hostname_pool_docker.yml`
(Star Wars planet names, independent of the Tart provider's pool). Append new
names to extend the pool. Names are allocated sequentially; no restart
needed.

## Default behavior unchanged

Omitting `provider` (or passing `-e provider=tart`) continues to use the Tart
provider exactly as before — see the top-level [README](../../README.md) /
`specs/009-create-destroy-vm/quickstart.md` for the Tart workflow.
