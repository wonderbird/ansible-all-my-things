# Tasks: Docker Provider for Create and Destroy VM Playbooks

**Input**: Design documents from `specs/011-docker-vm-provider/`
**Prerequisites**: plan.md, spec.md, research.md, data-model.md, contracts/

**Organization**: Grouped by user story for independent implementation and testing.
No test tasks — not requested in the specification.

## Format: `[ID] [P?] [Story] Description`

- **[P]**: Can run in parallel (different files, no shared dependency)
- **[Story]**: User story this task belongs to (US1 = Create Docker VM,
  US2 = Destroy Docker VM, US3 = Tart default unchanged)

## Phase 3: User Story 1 — Create a local Docker container as a "VM" (Priority: P1) 🎯 MVP

**Goal**: `create-vm.yml -e provider=docker` builds the image if needed,
starts a new privileged systemd container with a dynamically published SSH
port, registers it in `inventories/docker_autogenerated.yml`, and prints the
assigned hostname.

**Independent Test**: Run
`ansible-playbook playbooks/create-vm.yml -e provider=docker` with no other
extra-vars; verify the command completes, prints a hostname, and the container
is reachable via SSH using the registered inventory entry.

- [ ] T006 [US1] Create `playbooks/tasks/create/docker.yml` — pool-check
  block: load `playbooks/vars/hostname_pool_docker.yml` and
  `playbooks/vars/docker_credentials.yml` (`include_vars`); assert
  `docker_root_password | length > 0`; check for
  `inventories/docker_autogenerated.yml` via `stat`, load it with
  `include_vars` if present, else initialize an empty
  `all`/`linux`/`docker` `hosts: {}` skeleton via `set_fact` (mirrors
  `playbooks/tasks/create/tart.yml:38-58`); compute `used_hostnames` from
  `current_inventory.all.hosts.keys()`; fail loudly with a message naming
  `playbooks/vars/hostname_pool_docker.yml` if
  `(hostname_pool | difference(used_hostnames)) | length == 0` (FR-003,
  FR-007, Principle XII) **before any Docker action**; set `hostname` to the
  first unused pool entry
- [ ] T007 [US1] Extend `playbooks/tasks/create/docker.yml` — image build
  guard: run `docker image inspect ansible-vm-docker:latest`
  (`ansible.builtin.command`, `register: docker_image_check`,
  `changed_when: false`, `failed_when: false`); run
  `docker build -t ansible-vm-docker:latest --build-arg
  DOCKER_ROOT_PASSWORD={{ docker_root_password }} {{ playbook_dir }}/files/docker`
  `when: docker_image_check.rc != 0`, `changed_when: true` (FR-013,
  research.md "Docker Image Build Guard")
- [ ] T008 [US1] Extend `playbooks/tasks/create/docker.yml` — container-run
  block (wrapped in `block:` for the rescue in T009): set
  `docker_vm_cpus`/`docker_vm_memory` defaults (`2`/`4g`) if not already
  provided as extra-vars (FR-012); run
  `docker run -d --privileged --name {{ hostname }} --hostname {{ hostname }}
  --cpus {{ docker_vm_cpus }} --memory {{ docker_vm_memory }}
  -p 127.0.0.1::22 ansible-vm-docker:latest` (FR-002, FR-014); discover the
  published port via `docker port {{ hostname }} 22/tcp`
  (`register: docker_port_result`, `changed_when: false`); parse the host port
  from `docker_port_result.stdout` (format `127.0.0.1:<port>`) and assert it
  is non-empty/numeric before use (FR-005, Principle XII, research.md
  "Container Run, Port Discovery, and SSH Readiness")
- [ ] T009 [US1] Extend `playbooks/tasks/create/docker.yml` — SSH readiness
  and inventory write (still inside the T008 `block:`): poll SSH readiness on
  `127.0.0.1:<discovered-port>` via `ansible.builtin.wait_for_connection` (or
  `wait_for: port=<port> host=127.0.0.1`), mirroring the `tart ip` poll in
  `playbooks/tasks/create/tart.yml:105-129`; build the updated inventory dict
  via `combine(recursive=true)` adding the new host to `all.hosts` (with
  `ansible_host: 127.0.0.1`, `ansible_port: <discovered-port>`,
  `ansible_user: root`, `ansible_ssh_pass: docker_root_password`,
  `ansible_ssh_common_args` disabling host-key checking), `linux.hosts`, and
  `docker.hosts` (FR-004, FR-005, data-model.md "Docker Inventory Entry");
  write back to `inventories/docker_autogenerated.yml` via `copy` with
  `content: "{{ updated_inventory | to_nice_yaml(indent=2) }}"` and the
  `# AUTO-GENERATED` header comment (mirrors
  `playbooks/tasks/create/tart.yml:160-164`); emit a `debug` task printing
  `"VM created with hostname: {{ hostname }}"` (FR-006)
- [ ] T010 [US1] Extend `playbooks/tasks/create/docker.yml` — add the `rescue:`
  block for the T008 `block:`: run `docker stop {{ hostname }}` and
  `docker rm {{ hostname }}` (both `ansible.builtin.command`,
  `failed_when: false`), then `ansible.builtin.fail` with a message naming the
  hostname and stating that partial container state has been cleaned up
  (FR-015, mirrors `playbooks/tasks/create/tart.yml:170-186`)

**Checkpoint**: US1 complete —
`create-vm.yml -e provider=docker` functional and manually testable.

## Phase 4: User Story 2 — Destroy a Docker-provider VM by hostname (Priority: P2)

**Goal**: `destroy-vm.yml -e provider=docker -e hostname=<name>` stops and
removes the container (or warns and skips if already absent) and removes all
references from `inventories/docker_autogenerated.yml`.

**Independent Test**: Given a running Docker-provider container `tatooine`
(created via US1), run
`ansible-playbook playbooks/destroy-vm.yml -e provider=docker -e hostname=tatooine`;
verify the container no longer exists and `tatooine` is absent from
`inventories/docker_autogenerated.yml`.

- [ ] T011 [US2] Create `playbooks/tasks/destroy/docker.yml` — inventory check
  block: assert `hostname is defined and hostname | length > 0` (mirrors
  `playbooks/tasks/destroy/tart.yml:8-13`); check for
  `inventories/docker_autogenerated.yml` via `stat`, load it with
  `include_vars` if present, else initialize the empty
  `all`/`linux`/`docker` skeleton; fail loudly with an actionable error
  identifying the unknown hostname if `hostname` is not in
  `current_inventory.all.hosts` **before any Docker action** (FR-010,
  Principle XII, data-model.md "Docker Inventory Entry" validation rules)
- [ ] T012 [US2] Extend `playbooks/tasks/destroy/docker.yml` — container
  existence check and teardown: run `docker container inspect {{ hostname }}`
  (`ansible.builtin.command`, `register: docker_container_check`,
  `changed_when: false`, `failed_when: false`); when
  `docker_container_check.rc != 0`, emit a `debug` warning that the container
  is absent from Docker and the stop/remove steps are being skipped (stale
  entry, FR-009); when `docker_container_check.rc == 0`, run
  `docker stop {{ hostname }}` and `docker rm {{ hostname }}`
  (`changed_when: true`) (mirrors `playbooks/tasks/destroy/tart.yml:45-68`,
  substituting `docker container inspect` for the Tart `stat` check per
  research.md "Destroy Flow: Stale Inventory / Stale Container Handling")
- [ ] T013 [US2] Extend `playbooks/tasks/destroy/docker.yml` — inventory
  cleanup: rebuild `all.hosts`, `linux.hosts`, and `docker.hosts` via
  `dict2items | selectattr('key', '!=', hostname) | list | items2dict`
  (mirrors `playbooks/tasks/destroy/tart.yml:70-88`); write back to
  `inventories/docker_autogenerated.yml` via `copy` with
  `content: "{{ cleaned_inventory | to_nice_yaml(indent=2) }}"` and the
  `# AUTO-GENERATED` header comment; assert `hostname` is absent from
  `cleaned_inventory.all.hosts` afterwards (FR-009, Principle XII, mirrors
  `playbooks/tasks/destroy/tart.yml:96-99`)

**Checkpoint**: US2 complete —
`destroy-vm.yml -e provider=docker -e hostname=<name>` functional and
manually testable independently of US1 (given a pre-existing inventory entry).

## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies — start immediately
- **Foundational (Phase 2)**: No dependency on Phase 1 (Phase 2 only edits
  `create-vm.yml` / `destroy-vm.yml`); can run in parallel with Phase 1, but
  BLOCKS end-to-end testing of US1/US2
- **US1 (Phase 3)**: Depends on Phase 1 (pool, credentials, Dockerfile) and
  Phase 2 (dispatch)
- **US2 (Phase 4)**: Depends on Phase 1 (credentials/skeleton shape) and
  Phase 2 (dispatch); independent of US1's task file, but its independent test
  needs a container created by US1
- **US3 validation (Phase 5)**: Depends on Phase 2 (dispatch implemented) and
  benefits from US1/US2 existing for contrast
- **Polish (Phase 6)**: Depends on Phases 3, 4, and 5 being complete

### User Story Dependencies

- **US1 (P1)**: Requires Phase 1 + Phase 2 complete
- **US2 (P2)**: Requires Phase 1 + Phase 2 complete; can proceed in parallel
  with US1 (different file: `tasks/destroy/docker.yml` vs
  `tasks/create/docker.yml`), though its independent test consumes a US1
  output
- **US3 (P3)**: Requires Phase 2 complete only

### Within Each User Story

- T006 → T007 → T008 → T009 → T010 (sequential, same file)
- T011 → T012 → T013 (sequential, same file)
- T014 → T015 (sequential — T015 destroys the host T014 created)

### Parallel Opportunities

- T001, T002, T003 (Phase 1) can run in parallel — different files
- T004 and T005 (Phase 2) can run in parallel — different files
- Phase 1 and Phase 2 can run in parallel (no shared files)
- T006–T010 (US1) and T011–T013 (US2) can run in parallel once Phase 1 +
  Phase 2 are complete — different files
- T016–T020 (Phase 6) can run in parallel — independent validations

## Implementation Strategy

### MVP (User Story 1 Only)

1. Complete Phase 1 (T001–T003)
2. Complete Phase 2 (T004–T005)
3. Complete Phase 3: US1 (T006–T010)
4. **STOP and VALIDATE**:
   `ansible-playbook playbooks/create-vm.yml -e provider=docker`

### Full Delivery

1. Complete Phase 1 → Phase 2
2. Complete Phase 3 (US1) and Phase 4 (US2) — can overlap
3. Complete Phase 5 (US3 validation)
4. Complete Phase 6 (Polish & edge cases)
5. Run `quickstart.md` end-to-end (T021)

### Parallelism Note

Phase 4 (US2: destroy/docker.yml) is independent of Phase 3 (US1:
create/docker.yml) at the file level. After Phase 1 + Phase 2 complete, a
second implementer can begin T011 immediately while the first works on
T006–T010 — though T011's *independent test* requires a container created by
T006–T010 (or a manually-created inventory entry) to destroy.
