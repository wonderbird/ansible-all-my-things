# Quickstart: hcloud Provider for create-vm.yml / destroy-vm.yml

## Prerequisites

- `HCLOUD_TOKEN` exported in your shell (Hetzner Cloud API token).
- Your SSH private key matching `my_ssh_key_name` (vault-configured in
  `inventories/group_vars/all/vault.yml`) loaded in your SSH agent â€” this key
  must already be registered with Hetzner Cloud under that name.
- `requirements.yml` collections installed (`hetzner.hcloud` already
  declared).
- Vault password available via `ANSIBLE_VAULT_PASSWORD_FILE`.

## Create a disposable Hetzner Cloud server

```bash
ansible-playbook playbooks/create-vm.yml -e provider=hcloud
```

- Picks the next unused hostname from
  `playbooks/vars/hostname_pool_hcloud.yml`
  (`rivendell`, `hobbiton`, `bree`, `edoras`, `minas-tirith`, `mordor`,
  `isengard`, `moria`, `lothlorien`, `helms-deep`).
- Creates the server with the default type/location/image (overridable via
  `-e hcloud_server_type=...`, `-e hcloud_server_location=...`,
  `-e image=...`).
- On success, prints `VM created with hostname: <name>` and adds an entry to
  `inventories/hcloud_autogenerated.yml`.

Verify it's reachable:

```bash
ansible -i inventories/hcloud_autogenerated.yml <name> -m ping
```

## Destroy it

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=hcloud -e hostname=<name>
```

- Deletes the Hetzner Cloud server.
- Removes its entry from `inventories/hcloud_autogenerated.yml`.
- Removes its SSH host key from `inventories/hcloud_known_hosts`.

## Existing providers â€” unaffected

```bash
ansible-playbook playbooks/create-vm.yml                    # provider=tart (default), unchanged
ansible-playbook playbooks/create-vm.yml -e provider=docker  # unchanged
```

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `Required variable 'HCLOUD_TOKEN' ...` (before any API call) | `HCLOUD_TOKEN` unset/empty | `export HCLOUD_TOKEN=...` |
| `Hcloud hostname pool exhausted...playbooks/vars/hostname_pool_hcloud.yml` (before any API call) | All 10 pool names already in `inventories/hcloud_autogenerated.yml` | Destroy unused servers, or append names to the pool file |
| `Hostname '<x>' is not present in inventories/hcloud_autogenerated.yml` | Destroying a hostname that was never created (or already destroyed) | Use a valid hostname from `inventories/hcloud_autogenerated.yml` |
| `REMOTE HOST IDENTIFICATION HAS CHANGED` on create/connect | A *different* server now answers at a previously-known IP, and the stale key was never removed (e.g. a create that bypassed `destroy-vm.yml`) | This is the intended fail-loud MITM protection (ADR-003) â€” manually remove the stale line from `inventories/hcloud_known_hosts` only if you can confirm the IP reassignment is legitimate |
