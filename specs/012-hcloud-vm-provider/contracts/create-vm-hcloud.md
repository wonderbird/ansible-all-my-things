# Contract: `create-vm.yml -e provider=hcloud`

## Invocation

```bash
ansible-playbook playbooks/create-vm.yml -e provider=hcloud \
  [-e hcloud_server_type=<type>] [-e hcloud_server_location=<loc>] [-e image=<image>]
```

## Preconditions (checked in this order, before any Hetzner Cloud API call)

1. `provider` is one of `tart`, `docker`, `hcloud` (existing
   `tasks/assert-provider.yml`, extended).
2. `HCLOUD_TOKEN` environment variable is set and non-empty (FR-012). Failure
   message identifies `HCLOUD_TOKEN` by name.
3. At least one entry in `playbooks/vars/hostname_pool_hcloud.yml`
   (`hostname_pool`) is not already a key under `all.hosts` in
   `inventories/hcloud_autogenerated.yml` (FR-007). Failure message names
   `playbooks/vars/hostname_pool_hcloud.yml`.

## Inputs

| Variable | Required | Default | Source |
|---|---|---|---|
| `provider` | yes | — | extra-var |
| `HCLOUD_TOKEN` | yes | — | environment |
| `hcloud_server_type` | no | `hcloud_default_type` | `inventories/group_vars/hcloud_linux/vars.yml` |
| `hcloud_server_location` | no | `hcloud_default_location` | `inventories/group_vars/hcloud/vars.yml` |
| `image` | no | `ubuntu-24.04` | literal default |
| `my_ssh_key_name` | no (vault-provided) | — | `inventories/group_vars/all/vars.yml` (vault) |

## Effects on success

1. Exactly one new Hetzner Cloud server is created (FR-002) with `name` =
   the next available hcloud hostname, the resolved `server_type`,
   `location`, `image`, and `ssh_keys: [my_ssh_key_name]` (FR-013).
2. `inventories/hcloud_known_hosts` has any stale entry for the new server's
   IP removed (idempotent `ssh-keygen -R`, FR-005).
3. `wait_for_connection` succeeds against the new server's public IPv4 on
   port 22 as `root`, with
   `ansible_ssh_common_args: "-o StrictHostKeyChecking=accept-new -o UserKnownHostsFile=inventories/hcloud_known_hosts"`.
4. `inventories/hcloud_autogenerated.yml` gains one entry under `all`,
   `linux`, and `hcloud` groups for the new hostname, with `ansible_host`
   (public IPv4), `ansible_port: 22`, `ansible_user: root`, and the
   `ansible_ssh_common_args` from step 3 (FR-004, FR-005).
5. The assigned hostname is printed (FR-006).

## Effects on failure (after server creation, before completion)

- The partially-created Hetzner Cloud server is deleted
  (`hetzner.hcloud.server: state: absent`) before the playbook fails
  (FR-014).
- `inventories/hcloud_autogenerated.yml` and `inventories/hcloud_known_hosts`
  are left unchanged (no entry was written yet).

## Out of scope / unaffected

- `provider` omitted or `tart`/`docker`: behaves exactly as before this
  feature (FR-001, FR-016).
- `inventories/hcloud.yml` (dynamic inventory plugin),
  `provisioners/hcloud-linux.yml`, `destroy.yml`,
  `destroy-hcloud-tasks.yml`, and the `hobbiton` developer server: not read,
  written, or affected (FR-015).
