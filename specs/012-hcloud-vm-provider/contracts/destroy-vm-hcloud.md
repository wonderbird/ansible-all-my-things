# Contract: `destroy-vm.yml -e provider=hcloud -e hostname=<name>`

## Invocation

```bash
ansible-playbook playbooks/destroy-vm.yml -e provider=hcloud -e hostname=<name>
```

## Preconditions (checked in this order, before any Hetzner Cloud API call)

1. `provider` is one of `tart`, `docker`, `hcloud` (existing
   `tasks/assert-provider-profile.yml`, extended).
2. `hostname` extra-var is defined and non-empty (existing
   `destroy-vm.yml` pre_task, unchanged).
3. `hostname` is present as a key under `all.hosts` in
   `inventories/hcloud_autogenerated.yml` (FR-008, FR-010). If absent, fail
   with a clear error naming the unknown hostname — before any Hetzner Cloud
   action.

## Inputs

| Variable | Required | Default | Source |
|---|---|---|---|
| `provider` | yes | — | extra-var |
| `hostname` | yes | — | extra-var |

## Effects on success — server present in Hetzner Cloud

1. The Hetzner Cloud server named `hostname` is deleted
   (`hetzner.hcloud.server: state: absent`, FR-009).
2. Any host-key entry for that server's `ansible_host` (read from the
   inventory entry being removed) is removed from
   `inventories/hcloud_known_hosts` via idempotent
   `ssh-keygen -f inventories/hcloud_known_hosts -R <ip>` (FR-009).
3. The `hostname` entry is removed from `all`, `linux`, and `hcloud` groups in
   `inventories/hcloud_autogenerated.yml` (FR-009).

## Effects on success — server already absent from Hetzner Cloud (stale entry)

1. A warning is printed identifying `hostname` as stale (mirrors
   `tart`/`docker` stale-entry behavior).
2. The server-deletion API call is skipped.
3. Steps 2–3 above (known_hosts cleanup, inventory cleanup) still execute
   (FR-009) — no dangling entries remain.

## Postconditions (both paths)

- `hostname` is absent from `inventories/hcloud_autogenerated.yml`
  (asserted, mirrors `tart`/`docker`).
- No host-key entry for the removed server's former IP remains in
  `inventories/hcloud_known_hosts`.

## Out of scope / unaffected

- `provider` omitted or `tart`/`docker`: behaves exactly as before this
  feature (FR-001, FR-016).
- `inventories/hcloud.yml`, `provisioners/hcloud-linux.yml`, `destroy.yml`,
  `destroy-hcloud-tasks.yml`, and the `hobbiton` developer server: not read,
  written, or affected (FR-015).
